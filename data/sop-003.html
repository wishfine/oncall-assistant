前端Web On-Call SOP

文档编号：SOP-003 | 版本：v2.8 | 最后更新：2026-02-15

适用范围：前端Web团队全体值班人员，覆盖PC端&移动端H5&小程序WebView

一、值班职责

前端Web值班工程师是用户体验的最后守护者。值班人员需在收到告警后五分钟内响应,确认页面是否可正常访问,核心交互流程是否畅通。值班期间需要监控页面性能指标,包括首屏加载时间、白屏率、JavaScript错误率、接口调用成功率等。交接时需确认当前是否有活跃的前端故障,最近的前端发布记录,以及CDN配置变更情况。值班人员应熟练掌握Chrome DevTools、Lighthouse、Sentry等工具的使用,能够快速定位前端问题。每日早晨需检查前一晚的Sentry错误报告,清理已知的误报,标记需要跟进的新增异常。

二、监控指标

前端监控体系包括以下核心指标。性能指标方面：LCP(Largest Contentful Paint)需控制在2.5秒以内,FID(First Input Delay)需控制在100毫秒以内,CLS(Cumulative Layout Shift)需控制在0.1以内。这三项是Google Core Web Vitals的核心指标,直接影响SEO排名。稳定性指标方面：JavaScript错误率不超过千分之一,接口调用成功率不低于百分之九十九点五,资源加载失败率不超过千分之五。业务指标方面：页面PV/UV的分钟级波动,核心页面的跳出率,关键按钮的点击率。所有指标通过自研的前端监控SDK采集,上报到日志平台后通过Grafana展示。告警规则配置在AlertManager中,支持按页面、浏览器、地域等维度细分。

三、常见故障处理

场景一：白屏/页面无法加载

用户反馈页面白屏时,首先确认影响范围。通过监控平台查看白屏率指标,判断是全量用户还是部分用户受影响。如果是全量白屏,检查HTML主文档是否正常返回,CDN是否异常,DNS解析是否正确。如果是部分用户白屏,可能是JS文件加载失败,需要检查CDN节点状态和资源访问日志。常见原因包括：CDN证书过期导致HTTPS资源加载失败、新版本JS存在语法错误导致整个应用崩溃、接口返回异常数据导致渲染报错。紧急修复方案：如果是新发布导致,立即回滚到上一个版本;如果是CDN问题,切换到备用CDN或直接回源。

场景二：JS错误率突增

Sentry告警显示JS错误数量突增时,首先在Sentry中查看新增错误的堆栈信息和影响用户数。按浏览器版本、操作系统、页面路径等维度分析错误分布。如果错误集中在某个浏览器版本,可能是兼容性问题,需要检查是否使用了该浏览器不支持的API。如果错误集中在某个页面,检查该页面最近的代码变更。对于"Script error"类型的跨域脚本错误,需要确认静态资源服务器的CORS配置。对于"ChunkLoadError"类型的错误,通常是用户在版本更新后访问了旧版本缓存的页面,需要配置合理的缓存策略和版本回退机制。

场景三：页面加载性能下降

当LCP指标超过阈值时,通过监控平台分析性能下降的时间节点和影响范围。常见原因：一、打包体积过大,新功能引入了大量第三方库;二、图片未优化,大尺寸图片未使用WebP格式或未做懒加载;三、关键渲染路径阻塞,同步加载了非必要的CSS或JS资源;四、接口响应慢导致数据渲染延迟。排查工具：使用Lighthouse进行页面审计,使用Chrome Performance面板分析渲染时间线,使用webpack-bundle-analyzer分析包体积。临时优化方案：开启Gzip/Brotli压缩、启用HTTP/2 Server Push、对非核心模块做懒加载。

场景四：样式错乱/布局异常

收到样式错乱的反馈时,首先确认受影响的页面、浏览器和设备型号。检查CSS文件是否正常加载,是否存在CSS文件404的情况。如果是特定浏览器的兼容性问题,检查是否使用了该浏览器不支持的CSS属性,如CSS Grid在旧版IE中不支持。如果是移动端适配问题,检查viewport设置和响应式断点。常见原因还包括：CSS优先级冲突、第三方插件注入了全局样式、CSS-in-JS运行时错误导致样式未注入。紧急修复可通过CDN推送热补丁CSS文件覆盖问题样式。

场景五：接口跨域(CORS)错误

浏览器控制台出现CORS错误时,首先确认是哪个接口触发了跨域限制。检查接口服务的Access-Control-Allow-Origin响应头配置是否包含当前域名。如果是预检请求(OPTIONS)失败,检查服务端是否正确处理了OPTIONS方法。新增接口或接口域名变更时最容易出现此问题。临时解决方案：通过Nginx反向代理将接口请求转为同域请求。对于需要携带Cookie的跨域请求,需要同时设置Access-Control-Allow-Credentials为true,且Allow-Origin不能使用通配符*。

四、升级流程

前端值班遇到以下情况需升级：一、核心页面(首页、商品详情、下单页、支付页)全量白屏;二、JS错误率超过百分之一且持续上升;三、用户数据泄露风险(如XSS漏洞被利用);四、SEO核心页面无法被爬虫正常抓取。升级路径：值班前端→前端技术负责人→技术总监。P0故障需在五分钟内升级,同时在故障群内同步信息。

五、禁止操作

以下操作严格禁止：一、在生产环境直接修改CDN上的静态资源文件;二、在故障处理期间发布新版本(除非是紧急hotfix);三、未经代码审查直接修改线上配置;四、清除CDN全部缓存(应使用URL级别精确刷新);五、在HTML中直接内联大量JS代码绕过发布流程;六、修改全局公共组件而不通知其他业务线;七、在用户浏览器中执行eval或其他不安全的动态代码。

六、工具与命令参考

前端排障工具集：错误监控使用Sentry平台,支持Source Map还原生产环境堆栈。性能监控使用自研SDK,上报Core Web Vitals等关键指标。CDN管理使用内部CDN控制台,支持缓存刷新、预热、配置HTTPS证书。构建产物分析使用webpack-bundle-analyzer。兼容性查询使用Can I Use网站。移动端调试使用vConsole或Eruda工具。页面审计使用Lighthouse CLI,可集成到CI/CD流水线中。A/B测试使用内部实验平台,支持灰度发布和流量分配。
